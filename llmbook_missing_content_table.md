# Missing content, technology, library, discussion

| Missing area | Why it matters for a one-stop LLM/agents book | What exists now | What is missing / weak |
|---|---|---|---|
| **End-to-end agent systems architecture** | Engineers need a full reference architecture for real systems: planner, tool router, memory, execution sandbox, evaluator, recovery, permissions, cost control | Agents, tool use, multi-agent systems, safety, production are covered in separate chapters | Missing a single unifying “agent system blueprint” chapter tying all moving parts into deployable architectures |
| **Workflow orchestration and durable execution** | Production agents often depend on long-running workflows, retries, checkpointing, idempotency, human approvals, and resumability | Orchestration appears inside multi-agent/state management and production topics | No dedicated treatment of durable workflow engines, step semantics, async jobs, queues, cron/event triggers, compensation patterns |
| **Memory architecture in depth** | Researchers and engineers need a rigorous separation of context window, episodic memory, semantic memory, profile memory, retrieval memory, and write policies | Memory appears in conversational AI and agent foundations | Missing a consolidated chapter on memory taxonomy, storage design, write/read policies, forgetting, privacy, and evaluation |
| **Dataset engineering for applications** | Students and researchers need practical guidance on turning messy enterprise/user data into instruction, eval, preference, and tool-use datasets | Synthetic data, fine-tuning, alignment datasets are present | Missing applied dataset engineering for logs, conversations, tickets, knowledge bases, feedback loops, schema design, data contracts |
| **Open-source local deployment stack** | A one-stop book should help readers run serious local or self-hosted systems, not only API-based ones | Inference optimization, hardware, open-weight models, serving are present | Missing a practical local stack chapter: vLLM, SGLang, TGI, Ollama, llama.cpp, distributed inference tradeoffs, GPU sizing by use case |
| **Tooling ecosystem comparison by task** | Readers need concrete library choices for prompting, agents, eval, serving, orchestration, tracing, finetuning, and retrieval | Various tools are mentioned inside chapters such as LangSmith, Langfuse, Phoenix, Hydra, MLflow, W&B, promptfoo | Missing decision tables comparing major ecosystems by scenario: LangChain/LangGraph, LlamaIndex, DSPy, Haystack, OpenAI Agents SDK, AutoGen, CrewAI, PydanticAI, smolagents, etc. |
| **Reliability engineering for LLM apps** | Real systems fail in non-obvious ways: schema breakage, context poisoning, provider regressions, hidden latency spikes, non-deterministic edge cases | Evaluation, observability, production, safety cover parts of this | Missing a reliability chapter focused on failure taxonomy, incident response, rollback, canaries, fallback routing, SLOs/SLAs, chaos testing |
| **Research methodology for LLM papers** | Researchers and graduate students need help designing publishable studies, not just benchmarks | Evaluation chapter includes statistical rigor and reproducibility | Missing a chapter on research framing, contamination control, benchmark leakage, baselines, ablations, reviewer-proof experiment design, artifact release |
| **Human factors / UX for LLM systems** | Product builders need conversation design, trust signals, confidence display, intervention points, latency masking, editable provenance | Dialogue and frontend topics exist | Missing a serious UX chapter for LLM products and agents, including control surfaces, human override, audit visibility, and failure communication |
| **Enterprise integration patterns** | Corporate engineers need patterns for auth, RBAC, audit logs, secrets, connectors, document pipelines, policy enforcement | Strategy, compliance, production, tool design are covered | Missing concrete enterprise architecture patterns: identity integration, approval workflows, governance boundaries, tenant isolation, regulated deployment |
| **Advanced multimodal agents beyond generation** | Current systems increasingly combine text, vision, voice, documents, browser/computer use, and action loops | Multimodal generation, voice/interfaces, document retrieval, computer use agents are present | Missing a unified treatment of multimodal agent pipelines: perception → grounding → tool action → verification → memory |
| **Economic design of LLM systems** | Engineers and product leaders need cost-aware architecture at every layer | Cost/performance and ROI appear in a few chapters | Missing a consolidated systems-economics chapter: token budgeting, cascade design, routing, caching, eval cost, annotation cost, marginal cost vs quality |
