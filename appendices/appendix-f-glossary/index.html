Appendix F · Reference

Glossary of Key Terms

A comprehensive reference to the vocabulary of large language models, from attention heads to zero-shot learning

Every field invents its own language, then wonders why newcomers look confused. Consider this your phrasebook.

A Lexicographic AI Agent
How to Use This Glossary

Terms are organized alphabetically. Each entry includes a concise definition and a cross-reference to the section where the concept is discussed in depth. Use your browser's search function (Ctrl+F / Cmd+F) to jump to a specific term quickly.

A
Activation Function
A nonlinear mathematical function applied to the output of a neural network layer. Common choices include ReLU, GELU, and SiLU. Activation functions enable networks to learn complex, nonlinear relationships in data.
See: Section 0.2
Adapter
A small, trainable module inserted into a pretrained model to specialize it for a downstream task without updating the original weights. Adapters are a core technique in parameter-efficient fine-tuning (PEFT).
See: Section 14.1
Alignment
The process of training a language model to behave in ways that are helpful, harmless, and honest. Alignment techniques include RLHF, DPO, and constitutional AI. The alignment stage typically follows pretraining and supervised fine-tuning.
See: Section 16.1
Attention (Self-Attention)
A mechanism that allows each token in a sequence to attend to every other token, computing weighted relevance scores. Self-attention is the foundational building block of the transformer architecture and enables the model to capture long-range dependencies.
See: Sections 3.2, 4.1
Autoregressive Model
A model that generates output one token at a time, conditioning each new token on all previously generated tokens. GPT-family models and most modern LLMs use autoregressive decoding. This contrasts with masked language models like BERT.
See: Section 5.1
B
Backpropagation
The algorithm used to compute gradients of the loss function with respect to each parameter in a neural network. These gradients drive the weight updates during training via gradient descent.
See: Section 0.3
Batch Size
The number of training examples processed together in a single forward and backward pass. Larger batch sizes improve hardware utilization but require more memory. Gradient accumulation can simulate larger effective batch sizes.
See: Section 13.3
Beam Search
A decoding strategy that maintains multiple candidate sequences (beams) at each step, expanding and pruning them based on cumulative probability. Beam search tends to produce more coherent but sometimes less creative outputs than sampling methods.
See: Section 5.2
BERT (Bidirectional Encoder Representations from Transformers)
A pretrained encoder-only transformer model introduced by Google in 2018. BERT uses masked language modeling and is widely used for classification, NER, and other understanding tasks. It processes text bidirectionally, unlike autoregressive models.
See: Section 7.1
BFloat16 (Brain Float 16)
A 16-bit floating-point format that preserves the dynamic range of 32-bit floats while using less memory. Widely used during LLM training and inference on modern GPUs (A100, H100) because it reduces memory consumption with minimal accuracy loss.
See: Section 8.2
BitsAndBytes
A library that provides efficient quantization kernels for transformers, enabling 4-bit and 8-bit inference and training on consumer GPUs. It is the backend for QLoRA fine-tuning.
See: Section 14.2
C
Causal Language Modeling
A pretraining objective in which the model predicts the next token given all preceding tokens. A causal attention mask prevents the model from attending to future positions. This is the standard training objective for GPT-style models.
See: Section 6.1
Chain-of-Thought (CoT)
A prompting strategy that instructs the model to produce intermediate reasoning steps before arriving at a final answer. CoT significantly improves performance on arithmetic, logic, and multi-step reasoning tasks.
See: Section 10.2
Chatbot Arena
A crowd-sourced evaluation platform where human users compare model outputs side by side, generating Elo ratings. It provides one of the most reliable signals of real-world model quality because it captures human preference on open-ended tasks.
See: Section 25.2
Chinchilla Scaling Laws
Research by Hoffmann et al. (2022) showing that, for a fixed compute budget, model size and training data should be scaled roughly equally. This overturned earlier assumptions that favored larger models trained on less data.
See: Section 6.3
Chunking
The process of splitting documents into smaller segments for embedding and retrieval in RAG pipelines. Chunk size, overlap, and strategy (fixed-size, sentence-based, semantic) significantly affect retrieval quality.
See: Section 18.3
Classification
A task in which the model assigns one or more labels to an input. Sentiment analysis, spam detection, and topic categorization are common classification tasks. LLMs can perform classification via prompting or fine-tuning.
See: Sections 1.2, 11.1
Constitutional AI (CAI)
An alignment approach developed by Anthropic in which a model critiques and revises its own outputs according to a set of written principles (a "constitution"), reducing the need for human feedback at scale.
See: Section 16.4
Context Window (Context Length)
The maximum number of tokens a model can process in a single forward pass. Modern LLMs support context windows ranging from 8K to over 1M tokens. Longer contexts enable processing of entire documents but increase memory and compute requirements.
See: Section 7.2
Cosine Similarity
A metric that measures the angle between two vectors, yielding a value from -1 to 1. Cosine similarity is the standard metric for comparing text embeddings in semantic search and RAG systems.
See: Section 18.1
Cross-Entropy Loss
The standard loss function for language modeling. It measures the difference between the model's predicted token probability distribution and the true distribution (the actual next token). Lower cross-entropy indicates better predictions.
See: Section 6.1
D
Data Contamination
A situation in which benchmark or evaluation data appears in a model's training set, artificially inflating performance scores. Contamination is a growing concern as training corpora expand to encompass large portions of the internet.
See: Section 25.4
Decoding
The process of generating text from a language model by iteratively selecting the next token. Decoding strategies include greedy search, beam search, top-k sampling, nucleus (top-p) sampling, and temperature scaling.
See: Section 5.1
Distillation (Knowledge Distillation)
A technique for training a smaller "student" model to replicate the behavior of a larger "teacher" model. The student learns from the teacher's soft probability distributions rather than hard labels, transferring knowledge into a more efficient form.
See: Section 15.1
DPO (Direct Preference Optimization)
An alignment technique that trains models on pairs of preferred and rejected outputs without needing a separate reward model. DPO simplifies the RLHF pipeline by directly optimizing the policy from human preference data using a classification-style loss.
See: Section 16.3
Dropout
A regularization technique that randomly sets a fraction of neuron activations to zero during training, preventing co-adaptation. During inference, all neurons are active. Dropout helps prevent overfitting in neural networks.
See: Section 0.3
E
Embedding
A dense, fixed-dimensional vector representation of a token, sentence, or document in continuous space. Embeddings capture semantic meaning such that similar concepts map to nearby points. They are foundational to search, RAG, and classification systems.
See: Section 18.1
Encoder-Decoder Architecture
A transformer architecture with separate encoder and decoder stacks. The encoder processes the input, and the decoder generates the output while attending to the encoder's representations. T5 and BART are encoder-decoder models.
See: Section 4.3
Evaluation (LLM Evaluation)
The process of measuring model quality using benchmarks, human judgments, or automated metrics. Effective evaluation combines multiple approaches because no single metric captures all dimensions of language model performance.
See: Section 25.1
Evol-Instruct
A technique for generating increasingly complex instruction-tuning data by iteratively prompting an LLM to make existing instructions more difficult. First introduced in the WizardLM paper, it addresses the challenge of creating diverse, high-quality training examples.
See: Section 12.3
F
Feed-Forward Network (FFN)
A two-layer neural network applied independently to each position in a transformer layer, after the attention step. The FFN provides the model's main capacity for storing factual knowledge and performing computation.
See: Section 4.2
Few-Shot Learning
The ability of a model to perform a task given only a handful of examples in the prompt, without any weight updates. Few-shot learning is a hallmark capability of large language models and was prominently demonstrated by GPT-3.
See: Section 10.1
Fine-Tuning
The process of further training a pretrained model on a task-specific dataset. Fine-tuning updates the model's weights to specialize its behavior. Variants include full fine-tuning, LoRA, QLoRA, and other parameter-efficient methods.
See: Sections 13.1, 14.1
Flash Attention
An IO-aware attention algorithm that reduces memory usage from O(n²) to O(n) by tiling the attention computation and avoiding materialization of the full attention matrix. Flash Attention significantly speeds up both training and inference.
See: Section 8.1
FLOPS (Floating-Point Operations Per Second)
A measure of computational throughput. GPU specs typically list peak FLOPS in different precisions (FP32, FP16, BF16, INT8). FLOPS is a key metric for estimating training time and comparing hardware.
See: Appendix G
Function Calling (Tool Use)
A capability that allows LLMs to generate structured requests to external tools, APIs, or functions. The model outputs a JSON schema specifying which function to call and with what arguments. This is foundational for AI agents.
See: Sections 9.2, 21.2
G
GELU (Gaussian Error Linear Unit)
An activation function that smoothly gates inputs based on their magnitude. GELU is the default activation in most modern transformer architectures, including GPT and BERT families.
See: Section 4.2
GPT (Generative Pre-trained Transformer)
A family of autoregressive, decoder-only transformer models developed by OpenAI. Starting with GPT-1 (2018) through GPT-4 and beyond, the series demonstrated that scaling up language models yields increasingly capable general-purpose AI systems.
See: Section 7.1
Gradient Accumulation
A technique that sums gradients over multiple mini-batches before performing a weight update, effectively simulating a larger batch size. This is essential when the desired batch size exceeds available GPU memory.
See: Section 13.3
Grounding
Connecting a language model's outputs to factual, verifiable sources of information. RAG is the most common grounding technique. Grounding reduces hallucinations by anchoring responses in retrieved evidence.
See: Section 19.1
Grouped Query Attention (GQA)
An attention variant that shares key and value heads across groups of query heads, reducing the size of the KV cache while preserving most of the quality of multi-head attention. Used in Llama 2, Llama 3, and Mistral.
See: Section 4.1
H
Hallucination
A phenomenon in which a language model generates plausible-sounding but factually incorrect or fabricated information. Hallucination is an inherent challenge of autoregressive models because they optimize for fluency, not truth. Mitigation strategies include RAG, grounding, and self-consistency checks.
See: Sections 19.1, 26.3
HNSW (Hierarchical Navigable Small World)
A graph-based algorithm for approximate nearest neighbor search. HNSW builds a multi-layer graph that enables fast similarity search over millions of vectors. It is the default index type in many vector databases.
See: Section 18.2
HumanEval
A benchmark for evaluating code generation, consisting of 164 Python programming problems with unit tests. Models are scored on pass@k, the probability that at least one of k generated solutions passes all tests.
See: Section 25.2
I
Inference
The process of using a trained model to generate predictions or outputs for new inputs. Inference optimization is critical for production deployments and includes techniques like quantization, KV caching, batching, and speculative decoding.
See: Section 8.1
Instruction Tuning
A fine-tuning process that trains a model on (instruction, response) pairs to improve its ability to follow human directions. Instruction tuning bridges the gap between a raw pretrained model and a useful assistant.
See: Section 13.2
Interpretability (Mechanistic Interpretability)
The study of understanding how neural networks represent and process information internally. Techniques include attention visualization, probing classifiers, logit lens, and sparse autoencoders. The goal is to reverse-engineer the computations a model performs.
See: Section 17.1
J
JSON Mode (Structured Output)
A feature offered by LLM APIs that constrains the model to output valid JSON. This is essential for tool calling, data extraction, and any application that needs machine-parseable responses from an LLM.
See: Section 9.2
K
KV Cache (Key-Value Cache)
A memory optimization for autoregressive inference that stores previously computed key and value tensors so they do not need to be recomputed at each generation step. The KV cache is the primary memory bottleneck during long-context inference.
See: Section 8.1
Knowledge Graph
A structured representation of entities and their relationships, stored as (subject, predicate, object) triples. Knowledge graphs can augment RAG systems by providing structured context and enabling multi-hop reasoning.
See: Section 19.5
L
Layer Normalization
A normalization technique applied across the feature dimension of each token independently. It stabilizes training by keeping activations within a consistent range. Most modern transformers use pre-layer normalization (RMSNorm).
See: Section 4.2
LLM (Large Language Model)
A neural network with billions of parameters trained on vast text corpora using self-supervised learning. LLMs exhibit emergent capabilities such as in-context learning, reasoning, and instruction following. Examples include GPT-4, Claude, Gemini, and Llama.
See: Module 7
Logit
The raw, unnormalized score output by the final linear layer of a language model for each token in the vocabulary. Logits are converted to probabilities via the softmax function. The logit lens technique uses intermediate logits to inspect model internals.
See: Sections 5.1, 17.2
LoRA (Low-Rank Adaptation)
A parameter-efficient fine-tuning method that freezes the pretrained weights and adds small, low-rank decomposition matrices to each target layer. LoRA typically trains less than 1% of the original parameters while achieving performance close to full fine-tuning.
See: Section 14.1
M
MCP (Model Context Protocol)
An open protocol developed by Anthropic that standardizes how AI applications connect to external data sources and tools. MCP provides a universal interface so that agents and LLM-powered applications can access context from diverse systems.
See: Section 21.2
Multi-Head Attention (MHA)
An attention mechanism that runs multiple parallel attention computations (heads), each with its own learned projections. The outputs are concatenated and projected back. Multiple heads allow the model to attend to different types of relationships simultaneously.
See: Section 4.1
Mixture of Experts (MoE)
An architecture that replaces a single feed-forward network with multiple "expert" sub-networks and a gating mechanism that routes each token to a subset of experts. MoE enables larger total parameter counts while keeping inference cost manageable. Mixtral and DeepSeek-V3 use MoE.
See: Section 7.3
MMLU (Massive Multitask Language Understanding)
A benchmark consisting of 57 multiple-choice tasks spanning STEM, humanities, social sciences, and more. MMLU is widely used as a general knowledge metric for LLMs, though it has known limitations including data contamination concerns.
See: Section 25.2
Model Collapse
A degradation phenomenon in which models trained on synthetic data generated by earlier models progressively lose diversity and accuracy across generations. Model collapse is a key risk in synthetic data pipelines and motivates careful quality filtering.
See: Section 12.4
Model Merging
The practice of combining weights from multiple fine-tuned models into a single model without additional training. Merging methods include linear interpolation (LERP), SLERP, TIES, and DARE. It enables creating multi-skill models cheaply.
See: Section 15.3
N
Named Entity Recognition (NER)
A task that identifies and classifies named entities (persons, organizations, locations, etc.) in text. NER can be performed by fine-tuned encoders or by prompting LLMs with appropriate instructions.
See: Section 1.3
Next-Token Prediction
The core pretraining objective of autoregressive LLMs: given a sequence of tokens, predict the probability distribution over the next token. Despite its simplicity, this objective drives the acquisition of grammar, facts, reasoning, and other capabilities.
See: Section 6.1
Nucleus Sampling (Top-p Sampling)
A decoding strategy that samples from the smallest set of tokens whose cumulative probability exceeds a threshold p. Nucleus sampling dynamically adjusts the number of candidates, balancing diversity and coherence more effectively than fixed top-k.
See: Section 5.3
O
Observability
The practice of monitoring, logging, and tracing LLM system behavior in production. Observability tools capture inputs, outputs, latency, token usage, and error rates to enable debugging and continuous improvement.
See: Section 25.5
Overfitting
A condition in which a model performs well on training data but poorly on unseen data. During fine-tuning, overfitting manifests as memorization of training examples. It is mitigated by early stopping, regularization, and data augmentation.
See: Section 13.5
P
PEFT (Parameter-Efficient Fine-Tuning)
A family of techniques that fine-tune only a small subset of model parameters while keeping the rest frozen. PEFT methods (LoRA, QLoRA, adapters, prefix tuning) dramatically reduce memory and compute requirements compared to full fine-tuning.
See: Module 14
Perplexity
A metric that measures how well a language model predicts a sample. Mathematically, it is the exponentiated average cross-entropy loss. Lower perplexity indicates better prediction quality. Perplexity is commonly used to compare pretrained language models.
See: Section 6.2
Positional Encoding
A mechanism that injects position information into transformer inputs, since self-attention is inherently position-agnostic. Methods include sinusoidal encodings, learned embeddings, and rotary position embeddings (RoPE).
See: Section 4.4
Prefix Tuning
A PEFT technique that prepends trainable "virtual tokens" to the input at each transformer layer. Only these prefix parameters are updated, while the model weights remain frozen.
See: Section 14.1
Pretraining
The initial training phase in which a model learns general language representations from a large, unlabeled text corpus. Pretraining is computationally expensive (millions of GPU hours for frontier models) and produces a foundation model that can be adapted for specific tasks.
See: Section 6.1
Prompt Engineering
The practice of crafting inputs to an LLM to elicit desired outputs without changing model weights. Techniques include system prompts, few-shot examples, chain-of-thought, and structured output formatting.
See: Module 10
Prompt Injection
An attack in which adversarial text in the input causes the model to ignore its system instructions and follow attacker-supplied directives instead. Prompt injection is one of the most significant security challenges for LLM applications.
See: Section 27.1
Q
QLoRA (Quantized Low-Rank Adaptation)
A fine-tuning method that combines 4-bit quantization of the base model with LoRA adapters. QLoRA enables fine-tuning of 65B+ parameter models on a single consumer GPU with minimal quality loss.
See: Section 14.2
Quantization
The process of reducing the numerical precision of model weights (e.g., from 16-bit to 4-bit). Quantization reduces memory footprint and can accelerate inference with minimal quality degradation. Common formats include GPTQ, AWQ, and GGUF.
See: Section 8.2
R
RAG (Retrieval-Augmented Generation)
A technique that retrieves relevant documents from an external knowledge base and includes them in the LLM's context before generation. RAG grounds model responses in factual data, reducing hallucinations and enabling knowledge updates without retraining.
See: Module 19
Reasoning Model
A model specifically trained to perform extended step-by-step reasoning, often using a "thinking" phase before producing a final answer. Examples include OpenAI's o1/o3 and DeepSeek-R1. These models excel at math, science, and complex logic tasks.
See: Section 7.4
Reward Model
A model trained on human preference data to score the quality of LLM outputs. The reward model serves as a proxy for human judgment during RLHF training, providing the signal that guides the policy model toward better behavior.
See: Section 16.2
RLHF (Reinforcement Learning from Human Feedback)
An alignment technique that uses a reward model (trained on human preferences) to fine-tune an LLM via reinforcement learning (typically PPO). RLHF was a key breakthrough in making ChatGPT helpful and safe, but it is complex and expensive to implement.
See: Section 16.2
RMSNorm (Root Mean Square Normalization)
A simplified normalization technique that normalizes by the root mean square of activations, omitting the mean-centering step of standard layer normalization. RMSNorm is used in Llama, Mistral, and many modern architectures for its efficiency.
See: Section 4.2
RoPE (Rotary Position Embedding)
A position encoding method that applies rotation matrices to query and key vectors, encoding relative positions through the angle of rotation. RoPE naturally decays attention between distant tokens and supports context length extension.
See: Section 4.4
S
Scaling Laws
Empirical relationships showing that model performance improves predictably as a power law with increases in parameters, data, and compute. Scaling laws enable researchers to estimate the performance of larger models before training them.
See: Section 6.3
Semantic Search
Search based on the meaning of a query rather than keyword matching. Semantic search uses embeddings to find documents whose vector representations are close to the query vector. It is the retrieval backbone of most RAG systems.
See: Section 18.1
SFT (Supervised Fine-Tuning)
A training stage in which a pretrained model learns from labeled (input, output) pairs. SFT is the first post-pretraining stage, typically using high-quality instruction-response datasets. It is followed by alignment (RLHF or DPO).
See: Section 13.2
Softmax
A function that converts a vector of logits into a probability distribution, where each value is between 0 and 1 and all values sum to 1. Softmax is used in the attention mechanism and as the final layer of language models.
See: Section 4.1
Speculative Decoding
An inference optimization that uses a fast, small "draft" model to generate candidate tokens that are then verified in parallel by the larger target model. Accepted tokens skip individual decoding steps, yielding 2-3x speedups with no quality loss.
See: Section 8.3
Superposition
A phenomenon in which neural networks represent more features than they have dimensions by encoding features as nearly orthogonal directions in activation space. Superposition makes interpretability challenging because individual neurons may encode multiple concepts.
See: Section 17.3
SwiGLU
A gated activation function that combines the Swish activation with a gating linear unit. SwiGLU is used in many modern transformer architectures (Llama, Mistral, Gemma) because it consistently outperforms standard GELU or ReLU activations.
See: Section 4.2
Synthetic Data
Training data generated by AI models rather than collected from human sources. Synthetic data has become a critical resource for instruction tuning, alignment, and domain adaptation, but requires careful quality filtering to avoid model collapse.
See: Module 12
System Prompt
A special message at the beginning of a conversation that sets the model's role, behavior, constraints, and output format. System prompts are a primary mechanism for controlling LLM behavior in applications.
See: Section 10.1
T
Temperature
A scaling parameter applied to logits before softmax during text generation. Higher temperatures (e.g., 1.5) increase randomness and creativity; lower temperatures (e.g., 0.1) make outputs more deterministic and focused. A temperature of 0 is equivalent to greedy decoding.
See: Section 5.3
Token
The basic unit of text processed by a language model. Tokens can be words, subwords, or individual characters, depending on the tokenizer. A rough heuristic: one token is approximately 3/4 of an English word, or about 4 characters.
See: Section 2.1
Tokenizer
An algorithm that converts raw text into a sequence of integer token IDs from a fixed vocabulary. Common tokenizer algorithms include BPE (byte pair encoding), WordPiece, and SentencePiece. The tokenizer is a critical but often overlooked component of the LLM pipeline.
See: Module 2
Top-k Sampling
A decoding strategy that restricts the next-token selection to the k most probable tokens before sampling. Top-k prevents the model from selecting very unlikely tokens while maintaining diversity. Typical values range from 10 to 50.
See: Section 5.3
Transformer
A neural network architecture introduced in the 2017 paper "Attention Is All You Need." Transformers use self-attention and feed-forward layers to process sequences in parallel, replacing the sequential processing of RNNs. Virtually all modern LLMs are based on the transformer architecture.
See: Module 4
Transfer Learning
The practice of applying knowledge gained during pretraining on a large general corpus to downstream, task-specific problems. Fine-tuning a pretrained model is the most common form of transfer learning in NLP.
See: Section 13.1
U
Unsloth
An open-source library that provides optimized kernels for fine-tuning LLMs, achieving 2-5x speedups with 60% less memory compared to standard Hugging Face training. Unsloth is popular for QLoRA training of Llama, Mistral, and other models.
See: Section 14.2
V
Vector Database
A specialized database optimized for storing and searching high-dimensional vectors (embeddings). Vector databases support fast approximate nearest-neighbor search and are essential infrastructure for RAG systems. Examples include Pinecone, Weaviate, Qdrant, Chroma, and pgvector.
See: Section 18.2
vLLM
A high-throughput inference engine for LLMs that uses PagedAttention to manage GPU memory efficiently. vLLM supports continuous batching, tensor parallelism, and many quantization formats, making it a popular choice for production serving.
See: Section 8.4
W
Weight Decay
A regularization technique that adds a penalty proportional to the magnitude of model weights to the loss function. Weight decay helps prevent overfitting by discouraging very large parameter values. AdamW is the standard optimizer that decouples weight decay from gradient updates.
See: Section 13.3
Z
Zero-Shot Learning
The ability of a model to perform a task it was not explicitly trained for, using only a natural language description (no examples provided). Zero-shot capability is an emergent property of scale and is what makes LLMs useful as general-purpose tools.
See: Section 10.1
Zero-Shot Chain-of-Thought
A prompting technique that elicits reasoning by appending "Let's think step by step" to a zero-shot prompt, without providing any worked examples. This simple addition can dramatically improve performance on reasoning tasks.
See: Section 10.2

A Living Document

The LLM field evolves rapidly. New terms emerge with each major model release and research cycle. This glossary covers the core vocabulary needed to read and understand the material in this book. For the latest terminology, consult the original research papers and official documentation referenced in each chapter's bibliography.
